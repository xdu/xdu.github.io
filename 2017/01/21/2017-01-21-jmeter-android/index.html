追踪手机app的网路请求

2017-01-21

金融时报（FT）中文版app有一个非常有意思的功能，就是可以用双语显示同一篇报道。这个功能却不能在网页版中使用，我就很好奇，这个手机的app向服务器发出什么样的请求，能够获得双语的内容。

为了截获app的请求，我使用了jMeter（http://jmeter.apache.org）。打开jMeter，创建一个新项目，然后在WorkBench中添加以下的模块：

  • HTTP(S) Test Script Recorder
  • Recording Controller
  • View Results Tree

点击Recorder页面下的Start按钮，jMeter就创建一个监听8888端口的代理服务器；所有通过该代理的请求，以及服务器的应答都会被记录下来，下面要做的就是让手机使用这个代理，这样app做的网路请求就可以一览无遗了。

我使用Android手机，使用代理的方法如下：设定 -> Wi-Fi -> 选择当前使用的无线网路 -> Proxy (手动），在主机名称里输入jMeter运行的那台电脑的IP地址，比如说：192.168.0.3，端口输入填写8888，点击确定以使设置生效。

现在当手机运行“FT中文网”应用程序时，程序所做的网页请求就记录在jMeter里View Results Tree页面下。一个一个仔细分析后，就会发现应用程序请求了一个http://app003.ftmailbox.com/androidapp.html的网页。使用chrome打开这个网页，原来它就是“FT中文网”的入口，通过这个入口打开的报道都是用双语显示的。
