Jekyll

2016-05-06

Jekyll is a static site generation engine, it is so widely used today that no one can talk about the site generation without mention jekyll. The site generated by jekyll, for exmple a blog, can be hosted on the github for free, and the posts of the blog are stored in the plain text files instead of a database which makes the changes of the posts very handy. Jekyll can be used without internet connction, in total offline mode. Using the command line, you can even preview the site before publishing to the remote hosting server.

Jekyll uses the liquid templating language to customize the layout of the site, if you are not familiar with the web design, you can just pick up one theme from hundreds of the themes that you can find on the internet to start. After downloading a theme you like, just type bundle in the directory in which you uncompress the theme and all the software dependencies will be resolved and downloaded automatically. And then you can use the command line jekyll to start the preview site. The site contents, said the posts, are just the text file in markdown syntax, they are stored in the _posts directory.

The following commad lines create a new jekyll site using the theme ed. The last command starts the preview site.

$ git clone https://github.com/elotroalex/ed.git
$ cd ed
$ bundle install
$ jekyll serve

When you finish editing the site and are happy with the preview result, then you can upload to github site. To do that, just create a new repository with your username on github : username.github.io. it is important the follow this name convention or the site will not be published on github pages. Now upload all the files to this new repository and check this new jekyll site at username.github.io.
