React native setup

2016-05-29

React Native is a new player in the cross platform mobile development, it is based on the Facebook’s React library but specifically targets the writing of the applications on mobile phones, like iPhone or Android.

I followed the React Native Getting Started document on the projet home site for installation. The process is quite straightforward and has two versions for iphone and android developmenet environment respectively. As suggested by the document, I installed the command line tools : nodejs, npm, homebrew, watchman, flow on my notebook. For I want to start with an android application, so I downloaded and installed also the android SDK and the android studio. This is probably the most time consuming part of the installation, some extra packages of the SDK must be selected and downloaded to support the development, such as :

  • Android SDK Build-Tools 23.0.1
  • Google APIs
  • Intel x86 Atom System Image
  • Intel x86 Atom_64 System Image
  • Google APIs Intel x86 Atom_64 System Image

To end I set up the environment variable ANDROID_HOME and added the android tools library directory to PATH varible. Now I have all the development tool set, to run the first demo application I still need to setup an android virtual device. To create a virtual device is quite easy with the command line android avd. Right after the creation, I clicked the button start. A few seconds later, the android emulator device showed up.

With the react native project comes a example application ‘UIExplorer’ which demonstrates how to use the different UI components of the react native. This is a very good point to start to see how the react native works and to get familiar with the UI component. To compile this example application, I downloaded the source code of the react native and the android NDK version r10e. It is very important to get the right version of the NDK otherwise the compilation will fail. The compilation process is written with gradle and by the end of compilation the application will be installed to a virtual device or physical device. So before compilation, a android virtual device must be up and running. If it is not, use anroid avd.
